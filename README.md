# Artemis Financial – Secure Software Practices Report

## Client Summary and Software Requirements

**Client:** Artemis Financial  
Artemis Financial is a financial services company that specializes in managing and processing sensitive customer financial data. Because of the nature of its business, the company must comply with strict security and data integrity standards.

**Software Requirements and Issue Addressed:**  
The client requested a security review of its existing web-based software application. The main issue Artemis Financial wanted addressed was the presence of software security vulnerabilities that could expose sensitive data, compromise system integrity, or allow unauthorized access. The application required stronger protections such as secure communication, improved dependency management, and adherence to secure coding standards.

---

## Identifying and Addressing Software Security Vulnerabilities

**What I Did Well:**  
I effectively identified vulnerabilities by analyzing the application’s architecture, reviewing its dependencies, and evaluating areas where insecure practices could be exploited. I paid particular attention to data transmission, outdated libraries, and weak configurations.

**Why Secure Coding Is Important:**  
Secure coding reduces the risk of data breaches, financial loss, and reputational damage. Writing secure code helps prevent common attacks such as man-in-the-middle attacks, injection flaws, and dependency exploits.

**Value of Software Security to a Company:**  
Strong software security protects customer trust, ensures regulatory compliance, and minimizes operational risk. For a financial company like Artemis Financial, security directly impacts long-term stability and credibility.

---

## Vulnerability Assessment Experience

**Most Challenging or Helpful Aspect:**  
One of the more challenging aspects was interpreting vulnerability scan results and determining which findings posed real risk versus false positives. At the same time, this process was helpful because it improved my ability to prioritize vulnerabilities based on severity and impact.

## Increasing Layers of Security

**Security Enhancements Implemented:**  
- Added secure communication using encryption  
- Updated and managed third-party dependencies  
- Applied secure coding and configuration practices  
- Reduced the application’s attack surface through better validation and error handling  

**Future Assessment and Mitigation Approach:**  
In the future, I would continue using automated vulnerability scanning tools alongside manual code reviews. I would also rely on industry standards and risk assessment frameworks to determine the most effective mitigation techniques.

---

## Ensuring Functionality and Security

**Validation After Refactoring:**  
After refactoring the code, I verified that the application remained functional by testing core features and ensuring expected behavior was unchanged.

**Checking for New Vulnerabilities:**  
I re-ran vulnerability scans and reviewed the updated code to confirm that no new security issues were introduced during refactoring.
